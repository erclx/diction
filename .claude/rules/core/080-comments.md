# COMMENT STANDARDS

## Comments

- Do not add comments to source code, including `.ts`, `.tsx`, `.css`, and inline `<script>` in HTML. This covers JSDoc, intent, and section-header comments.
- Let clear naming and explicit structure carry the meaning. Extract a well-named constant or helper instead of explaining with a comment.
- This supersedes the intent-comment allowance in `.claude/rules/core/000-constitution.md`.
